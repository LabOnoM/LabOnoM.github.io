# Wiki Schema

To maintain consistency across all pages of the LLM-Wiki, follow these rules:

## 1. Directory Structure
- `/index.md`: Main directory index.
- `/overview.md`: Project summary.
- `/timeline.md`: Group/codebase milestones.
- `/log.md`: Wiki updates log.
- `/entities/`: Specific objects, pipelines, datasets, or analyses.
- `/sources/`: Raw documents, values, or reference markdown files.
- `/system/`: Tool summaries, environmental setups, and lessons learned.

## 2. Formatting
- Use Markdown headers hierarchically.
- Always include back-references: every entity must link back to `/index.md` and related entities/sources.
- Use Obsidian-style double bracket links (`[[page]]`) to link between pages.
